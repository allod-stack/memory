# NixOS Gotchas

## `/etc/nix/nix.conf` is read-only

It is a Nix store symlink. Write drop-ins to `/etc/nix/nix.conf.d/<name>.conf` or use the `NIX_CONFIG` environment variable. For bootstrap, `systemctl set-environment NIX_CONFIG="..."` persists until cleared; `nix.extraOptions` in configuration.nix makes it permanent after `nixos-rebuild`.

## `NIX_CONFIG` multi-line settings over SSH

Settings are newline-separated and should not be embedded in an SSH command string. Write a script:

```bash
ssh host 'tee /tmp/run.sh > /dev/null' << 'SCRIPT'
#!/bin/sh
export NIX_CONFIG='setting-one = value
setting-two = value'
exec the-actual-command
SCRIPT
ssh host 'bash /tmp/run.sh'
```

## `netrc-file` does not affect `git+https://` flake inputs

`netrc-file` in nix.conf only covers binary cache HTTP fetches. `git+https://` uses libgit2, which reads `~/.netrc` for the calling user. Three netrc files are needed:

- `/etc/nix/netrc` (640, root:users) - nix daemon via `netrc-file` in nix.conf
- `/root/.netrc` (600, root:root) - libgit2 when root runs nix
- `/home/${username}/.netrc` (600, user:users) - libgit2 when the user runs `nix flake update`

All three are generated by `profiles/modules/netrc.nix` from `/root/.git-credentials`.

## Experimental features in the NixOS installer

The minimal ISO may not have `nix-command` or `flakes` enabled. `nix.conf.d` drop-ins may not be picked up by CLI tools. Pass flags directly on every invocation:

```bash
nix --extra-experimental-features "nix-command flakes" run github:nix-community/disko ...
```

For `nixos-install`, use the `NIX_CONFIG` wrapper script pattern above.

## disko replaces `hardware-configuration.nix`

`nixos-generate-config` embeds machine-specific UUIDs. disko generates `fileSystems` at build time from a declarative `disk.nix`, so the same config works for any instance.

Add to flake.nix modules: `disko.nixosModules.disko` plus `./disk.nix`.

configuration.nix must absorb what `hardware-configuration.nix` used to provide:

```nix
{ config, lib, pkgs, modulesPath, platform, ... }: {
  imports = [ (modulesPath + "/profiles/qemu-guest.nix") ];
  boot.initrd.availableKernelModules = [ "ahci" "xhci_pci" "virtio_pci" "sr_mod" "virtio_blk" ];
  boot.kernelModules = [ "kvm-intel" ];
  nixpkgs.hostPlatform = lib.mkDefault platform;
}
```

For Allod VM configs, thread `platform` from inventory rather than hardcoding a Nix system string in profiles or checks.

## OpenSSH private key: never use `$()`

Command substitution strips the trailing newline, causing OpenSSH to reject the key with a libcrypto error. Always pipe directly:

```bash
age --decrypt secrets/host-key.age | ssh host 'sudo tee /etc/ssh/key > /dev/null'
```

## Never run commands that output SSH private key material

`ssh-keygen -y`, `cat`, or anything that reads a private key file sends it into the assistant transcript. To inspect a public key, read the `.pub` file. Private key files usually have no extension.

## agenix: declare secrets in flake.nix, not configuration.nix

`nixos-install` cannot activate secrets. `age.*` options in configuration.nix can fail at activation time. Put them in inline modules inside `nixosConfigurations` in flake.nix.

## QEMU VMs need UEFI, not SeaBIOS

systemd-boot requires UEFI. Set OVMF in virt-manager before first boot. It cannot be changed after VM creation.
